# Constitutional Runtime Restraint Standards

## Doctrine

Runtime restraint is the constitutional discipline that prevents readiness language from becoming runtime permission.

## Restraint Standards

Runtime governance must preserve:

- human review before activation;
- explicit scope;
- visible non-scope;
- adapter-only module interfaces;
- treasury separation;
- shared-core immutability;
- shutdown ability;
- rollback reviewability;
- audit preservation;
- fail-closed defaults.

## Prohibited Drift

Runtime restraint prohibits:

- implied activation;
- background execution;
- provider-triggered authority;
- adapter-triggered authority;
- model-generated approval;
- treasury side effects;
- hidden scope expansion;
- shared-core mutation.

## Constitutional Test

If a future reviewer cannot distinguish readiness from permission, restraint has failed.
