# Adapter Governance Stabilization Framework

## Purpose

Adapter governance stabilization ensures that adapters remain boundaries, not authority engines.

## Adapter Stabilization Requirements

Adapters must remain:

- scoped;
- explainable;
- replaceable;
- auditable;
- revocable;
- non-authorizing;
- provider-separated;
- module-contained;
- shared-core non-mutating.

## Governance Rule

Adapters may structure requests and responses. They may not approve actions, create execution authority, bypass human review, override treasury governance, mutate doctrine, or centralize orchestration.

## Instability Conditions

Adapter governance is unstable if an adapter can silently expand scope, hide provider behavior, translate ambiguity into approval, or make modules dependent on shared-core mutation.
